# Aura 拾光 · 色彩系统

> 色彩是产品情绪的第一载体。
> Aura 的色彩应该是 **温暖的、低饱和的、不疲劳的**。

---

## 一、色彩设计原则

### 1.1 核心理念

```
温暖的    而不是 冷冽的
低饱和的  而不是 鲜艳的
柔和的    而不是 对比强烈的
克制的    而不是 丰富的
```

### 1.2 色彩禁区

| 禁止 | 原因 |
|------|------|
| 高饱和色（S>60%） | 视觉疲劳、破坏安静感 |
| 纯黑 (#000000) | 过于生硬、不够温暖 |
| 纯白 (#FFFFFF) 作为背景 | 刺眼、缺乏质感 |
| 强对比配色 | 制造紧张感 |
| 渐变过渡突兀 | 视觉不适 |

## 二、品牌色

### 2.1 Aura Primary · 拾光橙

> 温暖的灵光，是品牌的核心识别色。

| 名称 | 色值 | HSL | 用途 |
|------|------|-----|------|
| `primary` | `#DF8020` | 30° 76% 50% | 品牌强调、按钮、进度条 |
| `primaryLight` | `#FFF7ED` | 30° 100% 96% | 轻量强调背景 |
| `primaryMuted` | `#FDBA74` | 30° 97% 72% | 柔和强调 |
| `primaryDark` | `#B86A1A` | 30° 76% 41% | 深色变体、按下态 |

**光效色（用于发光/阴影）**

| 名称 | 色值 | 用途 |
|------|------|------|
| `primaryGlow` | `#DF8020` @ 15% | 品牌光效 |
| `primaryGlowStrong` | `#DF8020` @ 25% | 强光效 |

**使用规范**

```
✅ 使用场景:
  - 主要按钮背景
  - 进度条/slider
  - 品牌标识
  - 关键操作入口

❌ 禁止场景:
  - 大面积背景
  - 长段文字
  - 与其他鲜艳色搭配
```

## 三、主背景色

### 3.1 Time Beige · 时间米白

> 像旧纸张、像晨光，是应用的主背景色。

| 名称 | 色值 | 用途 |
|------|------|------|
| `timeBeige` | `#FAF9F7` | App 全局主背景 |
| `timeBeigeLight` | `#FCFBF9` | 更浅的背景变体 |
| `timeBeigeWarm` | `#FDF9F5` | 温暖的背景变体 |

### 3.2 背景层级系统

| 层级 | 名称 | 色值 | 用途 |
|------|------|------|------|
| 基础层 | `bg` | `#FAF9F7` | 页面主背景 |
| 浮起层 | `bgElevated` | `#FFFFFF` | 卡片、弹窗背景 |
| 下沉层 | `bgMuted` | `#F5F4F2` | 次级背景区域 |

**层级使用示例**

```
页面背景 (bg)
  └── 卡片 (bgElevated)
        └── 卡片内次级区域 (bgMuted)
```

## 七、暗色模式

### 7.1 暗色模式原则

```
不是简单反转  而是重新设计明暗关系
保持温暖感    暗色背景也要有暖调
降低对比度    避免眩光
保持层级      仍然能区分前后层
```

### 7.2 暗色背景系统

| 层级 | 浅色模式 | 暗色模式 | 说明 |
|------|---------|---------|------|
| 基础背景 | `#FAF9F7` | `#18181B` | 页面背景 |
| 浮起背景 | `#FFFFFF` | `#27272A` | 卡片背景 |
| 下沉背景 | `#F5F4F2` | `#121214` | 次级区域 |

**暗色背景色值**

| 名称 | 色值 | 用途 |

| `bgDark` | `#18181B` | 主背景 |

| `bgMutedDark` | `#121214` | 下沉区域 |

### 7.3 暗色文字系统

### 7.6 情绪色适配

暗色模式下，情绪色整体降低明度：

| 颜色 | Light 变体 | 说明 |
|------|-----------|------|
| Calm Blue | `#1E3A5F` | 深蓝背景 |
| Warm Peach | `#4A2C1B` | 深桃背景 |
| Soft Green | `#1C3D1C` | 深绿背景 |
| Muted Violet | `#2D2640` | 深紫背景 |

### 7.7 功能色适配

功能色在暗色模式下保持相同，但背景使用更暗的变体：

| 颜色 | Light 背景 (浅色) | Light 背景 (暗色) |
|------|------------------|------------------|
| Success | `#DCFCE7` | `#14532D` |
| Warning | `#FEF9C3` | `#713F12` |
| Danger | `#FEE2E2` | `#7F1D1D` |

### 7.8 暗色模式实现

**Flutter 实现示例**

```dart
class AppColors {
  // 根据当前主题获取颜色
  static Color bg(BuildContext context) {
    return Theme.of(context).brightness == Brightness.dark
      ? const Color(0xFF18181B)
      : const Color(0xFFFAF9F7);
  }
  
  static Color textPrimary(BuildContext context) {
    return Theme.of(context).brightness == Brightness.dark
      ? const Color(0xFFFAFAFA)
      : warmGray900;
  }
  
  // ... 其他颜色
}

// 或使用 ColorScheme 扩展
extension AppColorScheme on ColorScheme {
  Color get auraBackground => brightness == Brightness.dark
    ? const Color(0xFF18181B)
    : const Color(0xFFFAF9F7);
}
```

### 7.9 暗色模式切换

```dart
支持模式:
  - 跟随系统（推荐默认）
  - 始终浅色
  - 始终深色

切换动效:
  - duration: 300ms
  - 整体渐变切换
```

## 八、色彩无障碍

### 8.1 对比度要求

| 元素 | 最低对比度 | 推荐对比度 |
|------|-----------|-----------|
| 正文文字 | 4.5:1 | 7:1 |
| 大文字（18px+） | 3:1 | 4.5:1 |
| 图标、边框 | 3:1 | 4.5:1 |

### 8.2 当前配色对比度

**浅色模式**

| 文字/背景组合 | 对比度 | 是否达标 |
|--------------|--------|---------|
| warmGray900 / timeBeige | 13.2:1 | ✅ |
| warmGray700 / timeBeige | 7.1:1 | ✅ |
| warmGray600 / timeBeige | 4.8:1 | ✅ |
| warmGray500 / timeBeige | 3.2:1 | ⚠️ 仅用于大文字 |
| warmGray400 / timeBeige | 2.1:1 | ❌ 仅用于装饰 |

### 8.3 色盲友好

```
原则:
  - 不仅依赖颜色传递信息
  - 错误/成功状态搭配图标
  - 情绪标签搭配文字
```
