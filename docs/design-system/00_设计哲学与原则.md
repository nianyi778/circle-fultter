# Aura 拾光 · 设计哲学与原则

> 本文档定义产品的核心设计理念，是所有设计决策的最高准则。
> 当设计出现分歧时，回到这里寻找答案。

---

## 一、产品定位

### 1.1 这是什么

**Aura 拾光** 是一个 **私密回忆时间胶囊**。

它帮助家庭（尤其是父母）：
- 记录孩子成长的点滴瞬间
- 为未来的自己和孩子留下真实的情绪痕迹
- 在时间流逝中，保存那些容易被遗忘的"当时"

### 1.2 它不是什么

| 它不是 | 为什么 |
|-------|--------|
| 效率工具 | 不追求"记录更多"，追求"记录有意义的" |
| 社交平台 | 不制造存在感，只提供共情空间 |
| 相册应用 | 不只是存图，而是保存"那一刻的状态" |
| 日记软件 | 不要求完整叙事，一句话也足够珍贵 |

### 1.3 产品气质

```
温柔的   而不是 热烈的
安静的   而不是 喧闹的
克制的   而不是 丰富的
缓慢的   而不是 高效的
真实的   而不是 精致的
```

## 二、核心设计原则

### 2.1 时间感优先

> 用户打开这个应用，第一感受应该是"时间"，而不是"功能"。

**具体表现：**
- 首页以"时间叙事"开场，而非功能入口
- 记录卡片必须包含时间语境（孩子年龄、相对时间）
- 所有日期显示采用人类可读格式（"3岁2个月"而非"2023-06-15"）

**自检问题：**
- 用户进入页面后，是否能立刻感知到"时间"？
- 回看内容时，是否能清晰还原"当时是什么时候"？

---

### 2.2 情绪安全

> 这个产品永远不应该制造焦虑、催促或压力。

**具体表现：**
- 不显示"未读"、"待办"、"N天未记录"等催促信息
- 不使用红点轰炸
- 空状态文案必须温暖，不责备
- 所有通知默认关闭，用户主动开启

**空状态文案原则：**
```
✗ "你还没有记录，快去记录吧！"
✓ "这里会慢慢被时间填满。"

✗ "去年今天没有内容"
✓ "去年今天，你还没有记录。等明年，它会出现的。"
```

**自检问题：**
- 用户看到这个界面，会感到焦虑吗？
- 用户不使用产品时，会有内疚感吗？

---

### 2.3 留白即呼吸

> 留白不是"没设计"，而是产品最重要的"情绪空间"。

**具体表现：**
- 模块之间必须有足够间距（≥24px）
- 页面不追求"一屏看完"，允许滚动
- 信息密度必须克制，宁少勿多
- 卡片内边距充足，文字不贴边

**自检问题：**
- 页面看起来拥挤吗？
- 用户是否有"喘息"的空间？

---

### 2.4 慢即仪式

> 在这个产品里，"慢"是功能，不是缺陷。

**具体表现：**
- 动画节奏偏慢（250-600ms）
- 重要操作有仪式感（封存信件、发布记录）
- 不追求操作效率，追求"动作的重量感"
- 页面切换不急促

**自检问题：**
- 这个操作是否有"完成感"？
- 用户是否感受到"这一刻被认真对待了"？

---

### 2.5 真实优于精致

> 记录的价值在于"真实"，不在于"好看"。

**具体表现：**
- 不提供复杂的图片滤镜
- 不鼓励修饰和美化
- 保留原始感（原图、原声、原样）
- 语境信息（当时的情绪）比内容修饰更重要

**自检问题：**
- 这个功能是在帮用户"变好看"还是"留下真实"？

---

## 三、反设计清单（禁止事项）

### 3.1 视觉禁区

| 禁止 | 原因 |
|-----|------|
| 高饱和色 | 破坏温柔感 |
| 强对比 | 制造视觉压力 |
| 粗边框 | 增加工具感 |
| 密集信息 | 产生认知负担 |
| 表格式布局 | 效率工具气质 |

### 3.2 交互禁区

| 禁止 | 原因 |
|-----|------|
| 弹跳动画 | 过于活跃，破坏安静感 |
| 抖动动画 | 制造焦虑 |
| 连续引导 | 产生催促感 |
| 强制教程 | 用户不是来"学产品"的 |
| 红点/角标 | 制造焦虑 |

### 3.3 文案禁区

| 禁止 | 原因 |
|-----|------|
| 催促语气 | "快去"、"马上"、"还没" |
| 数字攀比 | "你已经记录了N条" |
| 绩效语言 | "完成"、"达成"、"目标" |
| 游戏化语言 | "成就"、"勋章"、"等级" |

## 四、设计自检清单

在交付任何设计前，必须回答以下问题：

### 4.1 情绪检验

- [ ] 用户看到这个界面，会感到安心吗？
- [ ] 用户愿意在这里停留5秒以上吗？
- [ ] 用户不使用产品时，不会有内疚感吗？
- [ ] 界面是否没有任何焦虑元素？

### 4.2 时间检验

- [ ] 用户能感知到"时间"吗？
- [ ] 回看时能理解"那是什么时候"吗？
- [ ] 3年后再看这个设计，依然成立吗？

### 4.3 克制检验

- [ ] 信息密度是否足够低？
- [ ] 留白是否足够多？
- [ ] 动画是否足够慢？
- [ ] 功能是否足够少？

### 4.4 仪式检验

- [ ] 重要操作是否有"完成感"？
- [ ] 用户是否感受到"这一刻被认真对待"？

## 五、设计决策矩阵

当面临设计选择时，使用以下优先级：

```
情绪安全 > 时间感知 > 留白呼吸 > 仪式感 > 功能完整 > 操作效率
```

**示例决策：**

| 场景 | 选项A | 选项B | 选择 | 理由 |
|------|-------|-------|------|------|
| 首页信息量 | 展示6个模块 | 展示3个核心模块 | B | 留白 > 功能完整 |
| 删除确认 | 一键删除 | 二次确认+撤销 | B | 情绪安全 > 操作效率 |
| 动画时长 | 150ms快速 | 300ms缓慢 | B | 仪式感 > 操作效率 |
| 空状态 | "无内容" | "会慢慢填满的" | B | 情绪安全 |

## 七、设计语言一致性

### 7.1 用词规范

| 场景 | 使用 | 不使用 |
|------|------|--------|
| 发布内容 | 留下 | 发布、发送、提交 |
| 保存信件 | 封存 | 保存、提交、完成 |
| 点赞 | 共鸣 | 点赞、喜欢 |
| 返回 | 带走 | 返回、取消 |
| 继续编辑 | 留下来 | 继续、返回编辑 |

### 7.2 时间表达

| 场景 | 使用 | 不使用 |
|------|------|--------|
| 孩子年龄 | 3岁2个月 | 38个月、1156天 |
| 相对时间 | 2天前 | 2024-01-24 |
| 年度信标题 | 写给 TA · 4岁 | 2024年度信 |
| 去年今天 | 去年的今天 | 1年前的今天 |

---

> **记住：**
> 好的设计，不是让用户做更多事，
> 而是让用户意识到：时间正在发生。
